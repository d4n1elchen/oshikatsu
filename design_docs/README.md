# Design Documents for Oshikatsu

This directory contains the design documents for the Oshikatsu project. Each design document is organized into a folder named `[date]-[design-summary]`, detailing the planning and architecture for specific features or iterations.

## Structure

- `[date]-[design-summary]/`: Contains the detailed design, architecture, and implementation plan for a specific task or feature.

## Header convention

Each design doc begins with a short status block placed immediately after the title:

```markdown
> **Status:** <Landed | In progress | Proposed | Active roadmap>
> **Follow-ups:** <one-line note, or "—", or pointer to TECH_DEBTS.md>
```

Keep both lines short. Open work belongs in `TECH_DEBTS.md`, not in the design doc itself.
